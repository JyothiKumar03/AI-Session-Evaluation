*** PROBLEM STATEMENT
Build a Lightweight AI Workflow Evaluator Using Your Own Coding Transcripts 
At Slait, we care about how engineers work with AI, not just the final code they produce. 
For this take-home, you will build a small system that analyzes your own AI-assisted coding transcripts and produces structured insights about workflow quality. We want to see how you think, how you use AI tools, how you make tradeoffs, and how you turn an ambiguous problem into a clean, useful product. 
You should use the tools you normally use, such as Cursor, Claude Code, ChatGPT, Copilot, or similar. 
The Assignment 
Build a lightweight application that takes in one or more AI coding session transcripts and evaluates the quality of the engineering workflow shown in those transcripts. 
Important 
Please use your own real transcripts from coding sessions you have had with AI tools. These can come from: 
● Cursor 
● Claude Code 
● ChatGPT 
● Copilot 
● other AI coding assistants 
We are intentionally asking for real transcripts because the goal is to understand how you work with AI in practice.

● outputs structured insights

Product Goal 
Your tool should answer a question like: 
“How effectively is this engineer using AI during software development?” 
We are less interested in whether your scoring is “perfect” and more interested in whether your system is: 
● thoughtful 
● well-structured 
● grounded in useful signals 
● aligned with real engineering workflows 
Bonus Ideas 
Optional, but strong additions include:
● transcript comparison across multiple sessions 
● confidence score per metric 
● timeline / phase detection such as planning, implementation, debugging ● visual breakdown of workflow quality 
● support for different transcript formats 
● a short reflection on what makes an AI-assisted workflow “good”
